# 服务器监控预警系统说明

## 功能概述

已为现有的Netdata监控汇总平台添加了完整的告警和通知功能。

## 新增功能

### 1. 告警规则管理

- **预设快速指标**: CPU使用率、内存使用率、磁盘使用率、GPU使用率、CPU温度、GPU温度
- **自定义Netdata指标**: 支持从Netdata动态加载所有可用的Charts和Dimensions
- **告警条件**: 高于/低于指定阈值
- **告警级别**: 警告、严重
- **连续触发**: 支持设置连续触发次数，避免抖动
- **服务器范围**: 可应用到所有服务器或指定服务器
- **启用/禁用**: 可随时启用或禁用规则

### 2. 自定义Netdata指标配置

系统现在支持直接从Netdata服务器获取完整的指标列表：

1. 选择一个已配置Netdata的服务器
2. 系统自动加载该服务器所有可用的Charts（指标）
3. 选择Chart后显示该指标下的所有Dimensions（维度）
4. 支持搜索过滤，快速找到需要的指标
5. 自动获取指标单位（百分比、°C、MB/s等）

### 3. 通知渠道

支持以下通知方式：

- **Webhook**: 通用HTTP Webhook
- **企业微信**: 企业微信群机器人
- **钉钉**: 钉钉群机器人
- **邮件**: 通过Webhook发送邮件

### 4. 告警检测引擎

- 预设指标：每8秒自动检查（与Dashboard同步）
- 自定义Netdata指标：每30秒独立检查
- 支持连续触发次数配置
- 自动检测恢复并发送恢复通知
- 告警状态: pending -> firing -> acknowledged -> resolved

### 5. 告警历史和通知日志

- 完整的告警事件记录（触发、确认、恢复）
- 通知发送日志，包含成功/失败状态
- 支持按服务器筛选记录

## 使用指南

### 1. 添加预设指标告警规则

1. 点击左侧导航栏的「告警配置」
2. 选择「告警规则」标签页
3. 点击「+ 添加规则」按钮
4. 填写规则信息：
   - 规则名称
   - 应用服务器（可选，留空则应用到所有）
   - 监控指标（从预设列表选择：CPU/内存/磁盘/GPU/温度）
   - 触发条件（高于/低于）
   - 阈值
   - 告警级别
   - 连续触发次数（建议2-3次避免抖动）
5. 保存规则

### 2. 添加自定义Netdata指标告警规则

1. 点击左侧导航栏的「告警配置」
2. 选择「告警规则」标签页
3. 点击「+ 添加规则」按钮
4. 填写基本信息：
   - 规则名称
   - 应用服务器（**必须选择一个已配置Netdata的服务器**）
5. 在指标类型中选择「自定义Netdata指标」
6. 系统会自动加载该服务器的所有可用指标
7. 使用搜索框过滤找到需要的指标
8. 在左侧选择Chart（指标类别）
9. 在右侧选择Dimension（具体维度）
10. 配置触发条件、阈值、告警级别等
11. 保存规则

### 3. Netdata指标说明

- **Chart**: Netdata的指标类别，如 `system.cpu`、`mem.available`、`net.eth0` 等
- **Dimension**: 具体的指标维度，如 `user`、`system`、`idle`、`used`、`free` 等
- **Units**: 指标单位，自动从Netdata获取（%、°C、MB/s、kB等）

### 2. 配置通知渠道

1. 在「告警配置」页面选择「通知渠道」标签页
2. 点击「+ 添加渠道」按钮
3. 选择渠道类型并填写配置：
   - **Webhook**: 填写Webhook URL
   - **企业微信/钉钉**: 填写群机器人Webhook URL
   - **邮件**: 填写收件人邮箱和可选的邮件服务Webhook
4. 保存并启用渠道

### 3. 查看和处理告警

- **仪表盘**: 顶部显示活跃告警，可直接确认告警
- **服务器卡片**: 有告警的服务器会显示告警标记和数量
- **告警历史**: 查看所有历史告警事件
- **通知日志**: 查看通知发送记录

## Webhook 数据格式

当发送Webhook通知时，数据格式如下：

```json
{
  "title": "服务器告警 - 严重",
  "message": "[严重] server1 - CPU 使用率 95.5% 高于 90%",
  "alert": {
    "id": "alert-id",
    "serverName": "server1",
    "metric": "CPU 使用率",
    "value": 95.5,
    "threshold": 90,
    "condition": "above",
    "level": "critical",
    "status": "firing",
    "startedAt": "2024-01-01T00:00:00.000Z"
  },
  "timestamp": "2024-01-01T00:00:00.000Z"
}
```

## 默认建议配置

### 推荐告警规则

1. **CPU 使用率过高**
   - 指标: CPU使用率
   - 条件: 高于
   - 阈值: 90%
   - 级别: 警告
   - 连续: 2次

2. **内存使用率过高**
   - 指标: 内存使用率
   - 条件: 高于
   - 阈值: 85%
   - 级别: 警告
   - 连续: 2次

3. **磁盘使用率过高**
   - 指标: 磁盘使用率
   - 条件: 高于
   - 阈值: 90%
   - 级别: 严重
   - 连续: 1次

4. **CPU 温度过高**
   - 指标: CPU温度
   - 条件: 高于
   - 阈值: 80°C
   - 级别: 警告
   - 连续: 2次
